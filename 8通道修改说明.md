# 8通道布局修改说明

## 修改概述
将客户端实时检测管理页面从16个视频通道改为8个视频通道，采用新的布局设计：
- 左侧：任务表格（550px宽）
- 中间：大预览窗口（1200x900px，4:3比例）
- 右侧：8个小视频窗口（240x180px，4:3比例，垂直排列，带滚动条）

## 修改的文件

### 1. client/widgets/system_window.py
- **修改内容**：
  - 完全重写 `_createDefaultVideoLayout` 方法
  - 移除旧的滚动区域和2x8网格布局
  - 新增大预览窗口（1200x900px，4:3比例）
  - 右侧8个小视频窗口（240x180px，4:3比例）垂直排列
  - 右侧添加滚动区域（容器高度1475px > 显示高度900px）
  - 历史视频面板数量从 16 个改为 8 个
  - 曲线模式通道容器从 16 个改为 8 个
  - 所有 `range(16)` 改为 `range(8)`
  - 所有 `range(1, 17)` 改为 `range(1, 9)`

### 2. client/widgets/videopage/channelpanel.py
- **修改内容**：
  - `__init__` 方法新增 `width` 和 `height` 参数，支持自定义尺寸
  - `_initUI` 方法根据自定义尺寸或使用默认尺寸（620x465）
  - 使小视频面板能够适应240x180的尺寸（4:3比例）

### 3. client/handlers/videopage/channelpanel_handler.py
- **修改内容**：
  - 配置文件加载循环从 16 个通道改为 8 个通道
  - 配置保存循环从 16 个通道改为 8 个通道
  - 通道名称更新循环从 16 个通道改为 8 个通道
  - 所有 `range(1, 17)` 改为 `range(1, 9)`

### 4. client/widgets/videopage/missionpanel.py
- **修改内容**：
  - 新建任务通道复选框从 16 个改为 8 个
  - 通道调试按钮从 16 个改为 8 个
  - 所有 `range(1, 17)` 改为 `range(1, 9)`

### 5. client/handlers/videopage/missionpanel_handler.py
- **修改内容**：
  - WebSocket订阅通道列表从 channel1-channel16 改为 channel1-channel8
  - `range(1, 17)` 改为 `range(1, 9)`

### 6. client/ui/page0_video.ui
- **已有布局**（参考设计）：
  - 左侧：任务表格（550px宽）
  - 中间：大预览窗口 videoWidget_preview（1200x900px）
  - 右侧：8个小视频窗口 videoWidget_0 到 videoWidget_7（原设计240x107px）
  - **注意**：Python代码中已改为240x180px（4:3比例）以保持视频比例协调

### 视频面板尺寸
- **大预览窗口**：1200x900px（4:3比例）
- **小视频窗口**：240x180px（4:3比例，保持视频比例协调）
- **小窗口间距**：5px
- **右侧滚动区域**：260x900px（包含滚动条）
- **容器总高度**：240×8 + 5×7 = 1440 + 35 = 1475px
- **需要滚动**：1475px > 900px，因此添加垂直滚动条

## 功能特性

### 大预览窗口
- 用于显示选中通道的放大视频
- 可以点击小视频窗口切换预览内容
- 支持完整的视频控制功能

### 小视频窗口
- 显示8个通道的实时视频缩略图
- 4:3比例，保持视频不变形
- 垂直排列，带滚动条
- 点击可切换到大预览窗口
- 保留所有通道控制按钮（连接、设置、曲线、放大）

## 注意事项

1. **通道编号**：
   - 通道编号仍然是 1-8（channel1-channel8）
   - 不是 0-7

2. **预览窗口**：
   - 预览窗口是一个独立的ChannelPanel实例
   - 可以通过 `self.previewPanel` 访问
   - 需要实现点击小窗口切换预览的逻辑（后续开发）

3. **配置文件**：
   - 配置文件中的 channel9-channel16 配置不会被使用
   - 但保留这些配置不会影响系统运行
   - 如需清理，可以手动删除配置文件中的相关部分

4. **扩展性**：
   - 如果将来需要恢复16通道，只需修改相关循环和布局
   - ChannelPanel支持自定义尺寸，便于灵活布局
