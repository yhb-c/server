# MySQL Root权限配置和Go API数据库连接错误修复规范

## 介绍

用户报告了MySQL root用户权限配置和Go API数据库连接方面的问题。系统中存在多个不一致的数据库配置脚本，导致Go API无法正常连接到MySQL数据库。主要问题包括：密码配置不一致、数据库名称不统一、用户权限配置混乱等。

## 错误分析

### 当前行为（缺陷）

1.1 当Go API尝试连接数据库时，系统因为root密码不一致而连接失败（配置中使用'root'，某些脚本设置为'root123'）

1.2 当执行不同的数据库初始化脚本时，系统创建不同名称的数据库（'liquid_detection' vs 'liquid_db'），导致连接目标不一致

1.3 当系统尝试使用数据库连接时，系统因为用户权限配置混乱而出现认证错误（有些脚本创建专用用户'liquid_user'，有些直接使用root）

1.4 当Go API启动时，系统因为数据库配置与实际MySQL设置不匹配而无法建立连接

### 预期行为（正确）

2.1 当Go API尝试连接数据库时，系统应该使用统一的root密码'root'成功建立连接

2.2 当执行任何数据库初始化脚本时，系统应该统一创建和使用'liquid_detection'数据库

2.3 当系统进行数据库操作时，系统应该使用统一配置的root用户权限，确保认证成功

2.4 当Go API启动时，系统应该成功连接到MySQL数据库并显示"数据库连接成功"消息

### 不变行为（回归预防）

3.1 当数据库包含现有数据时，系统应该继续保持所有现有表结构和数据完整性

3.2 当Go API提供健康检查接口时，系统应该继续返回正确的数据库连接状态

3.3 当系统执行数据库查询操作时，系统应该继续保持现有的查询性能和结果准确性

3.4 当多个服务同时访问数据库时，系统应该继续维持正常的并发访问能力
